## üìã Documentation Standards

### File Naming Convention
- Use descriptive, kebab-case filenames
- Include relevant prefixes (e.g., `MOBILE_`, `API_`, `GUIDE_`)
- End with `.md` extension

### Organization Principles
- **Features**: Group by business domain/feature area
- **Technical**: Separate frontend, backend, architecture, deployment
- **Process**: Guides, testing, troubleshooting separate from implementation
- **Historical**: Archive old documentation but keep for reference

### Contributing to Documentation
1. Place new documentation in the appropriate category folder
2. Update this README.md if adding new major sections
3. Follow existing markdown formatting conventions
4. Include code examples where relevant
5. Add cross-references to related documentation
